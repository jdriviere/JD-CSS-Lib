Alerts

Types

Alerts can serve to make sure users can see important messages on their screen. Simply create a block of code and stylize it accordingly. Start off by adding an alert, and then specify which type of alert you want by adding the second class, alert--$type, where $type is either one of the following: default, info, success, warning, or danger.

This is a default alert
This is an info alert
This is a success alert
This is a warning alert
This is a danger alert
Code:
<div class="alert alert--$type">
// Body goes here...
</div>
Components

Alerts also contain more features, such as a title, description, and list--if you wish to make your alerts cleaner.

This is an alert description.

  • Second alert in the list
  • Third alert in the list
Code:
<div class="alert alert--$type">
<h4 class="alert__title">Alert Title</h4>
<p class="alert__desc">Some text description here</p>
<ul>
<li class="alert__list">Some alert list #1</li>
<li class="alert__list">Some alert list #2</li>
<li class="alert__list">Some alert list #3</li>
</ul>
</div>

Buttons

Base

One of the most essential elements in a CSS framework: buttons. Sparkle allows you to style anchor tags, div tags, and buttons by simply putting button in its class. This is also the first step to style a button accordingly (see other styling below).

Button
Button
Code:
<button class="button">Some Text</button>
<input type="button" value="Some Text" class="button">
<a href="#" class="button">Some Text</a>
<div class="button">Some Text</div>
Types (Solid)

Sparkle offers a variety of button colors, to go with your design needs. To specify the type of button you want, you must add button--$type to your button class, where $type is either one of the following: primary, secondary, success, warning, danger, light, or dark.

Code:
<button class="button button--$type">Some Text</button>
Types (Outlined)

If solid-colored buttons are not enough, Sparkle also offers the option of outlining the buttons. Simply add --outline at the end of the button type class, as in button--$type--outline.

Code:
<button class="button button--$type--outline">Some Text</button>
Types (Ghosted)

Sparkle also offers the "ghost" button variety too. Simply add --ghost at the end of the button type class, as in button--$type--ghost.

Code:
<button class="button button--$type--ghost">Some Text</button>
Sizes

Buttons come in different sizes for different purposes. You can alter a button size by adding button--$size to the class list, where the $size is either one of the following: xsmall, small, medium, large, or xlarge.

Code:
<button class="button button--$type[--$option] button--small">Some Text</button>
<button class="button button--$type[--$option] button--medium">Some Text</button>
<button class="button button--$type[--$option] button--large">Some Text</button>
<button class="button button--$type[--$option] button--xlarge">Some Text</button>
Block

You can also have your button take the entire span of the block too by adding button--block to your class list. You can even mix it with the button roundness classes (see below).

Code:
<button class="button button--$type[--$option] button--block">Some Text</button>
Roundness

Sparkle offers the chance to even vary the roundness of its button design. Simply add button--$roundness in your class list, where $roundness is one of the following: h-round (for 'half-round'), or f-round (for 'fully-round').

Code:
<button class="button button--$type[--$option] button--h-round>Some Text</button>
<button class="button button--$type[--$option] button--f-round>Some Text</button>
Disabled

You want to insert a button but not want it to be accessible yet? No problem! simply add button--disabled to your list of classes, and use a code to remove that class whenever you need.

Code:
<button class="button button--$type[--$option] button--$size button--disabled">Some Text</button>

Inputs

Input

Sparkle offers the possibility to stylize inputs. Just add input to the class list of the <input> you wish to stylize. You can also add input__label to the <label> tag, if you wish to add one.

Code:
<label for="input" class="input__label">Input:</label>
<input type="text" class="input" id="input" placeholder="Valid input">
Text Area

Add a input__textarea class to the <textarea> you wish to stylize.

Code:
<textarea class="input__textarea" placeholder="Type text here"></textarea>
Selection

Add a input__select class to the <select> tag you wish to stylize.

Code:
<select class="input__select">
<option value="valueA">Option 1</option>
<option value="valueB">Option 2</option>
...
</select>

Tables

Base

In Sparkle, the table feature already comes fully responsive. There is no need to style each of its elements. Simply add the table class to the <table> tag.

ID First Name Last Name Age
1 John Smith 24
2 Sally Johnson 29
3 Rupert Flint 26
Code:
<table class="table">
<thead>
<tr>
<th>ID</th>
<th>First Name</th>
<th>Last Name</th>
<th>Age</th>
</tr>
</thead>

<tbody> <tr>
<td data-label="ID">1</td>
<td data-label="First Name">John</td>
<td data-label="Last Name">Smith</td>

Typography

Base

The default font used in Sparkle is Google Font's Lato, with Helvetica Neue as its backup. The html font-size was reduced to 62.5% of its original size, so that the root EM units can be in multiples of 10. For example, a value of 1.2rem would be 12px, approximately. The root EM was set, by default, to 1.5rem, which is about 15px.

H1 (4.8rem)

H2 (4.2rem)

H3 (3.6rem)

H4 (2.8rem)

H5 (2.4rem)
H6 (2.0rem)

<h1>Some text</h1>
<h2>Some text</h2>
<h3>Some text</h3>
<h4>Some text</h4>
<h5>Some text</h5>
<h6>Some text</h6>
