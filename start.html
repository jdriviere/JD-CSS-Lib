What is Sparkle?

Sparkle is a lightweight, responsive, open-source CSS framework created to help Web Designers and Developers create simple, responsive websites and web applications. It was developed by J. Djimitry Riviere in late 2016, and continues to be improved upon by its developer. The major key to this framework lies within its ease of use to deliver superb results when designing apps and websites.

Why Choose Sparkle?

Because Sparkle is...

Semantic

Sparkle uses BEM to keep the framework structured and simple for all to understand and use.

User-Friendly

Sparkle does most of the hard work for you. Simply style your elements.

Responsive

Sparkle was created to be responsive throughout all devices, using a mobile-first approach.

Lightweight

Sparkle is smaller than 20kb, which means it won't slow down your website loading.

Let's Get Started

There are 4 ways to get started with the Sparkle framework:
1. Clone your own copy by using the following command in your command line:

git clone https://github.com/jdriviere/sparkle-css.git

(Note: If you wish to contribute to this project, be sure to read description below);

2. Download the .ZIP file;

3. Link the latest release directly into your HTML file by inserting the following lines to your HTML file:

<!-- Place this in your <head> section -->
<link rel="stylesheet" href="https://gitcdn.link/repo/jdriviere/sparkle-css/master/dist/css/sparkle.min.css" >
...
<!-- Place this at the end of your <body> section -->
<script src="https://gitcdn.xyz/repo/jdriviere/sparkle-css/master/dist/js/sparkle.min.css"></script>

(Note: You will need both links to ensure Sparkle works efficiently, as it uses the jQuery.)

4. Use the NPM/Bower command to download the package into your node_modules folder:

npm install sparkle.css

(Note: Node.js and NPM must be installed in your machine first.)
bower install sparkle-css

(Note: Node.js and Bower must be installed in your machine first.)

Contributing

You are encouraged to contribute to this project. Simply clone the repository, and come up with new features, bug fixes, etc. and modify the files. Make sure to follow the guidelines for the styling before posting.
